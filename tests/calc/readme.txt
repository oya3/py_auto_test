課題
電卓アプリの xpath が cp932 <---> utf8 に相互に変換することができない様子。
問題箇所は、１個目の"電卓"と"-"の間に不正な文字が含まれるっぽい。
emacs と xyzzy に xpath を確認したが両方に u200e の文字コードが含まれている。
変換できないのは、ruby の問題なのかもしれないが、おそらく windows application driver(ui recorder) の問題だと思われる。。。
で、結局 ruby で 外部utf8ファイルを読み込んで実行時に cp932 にもとに戻せないので
このままでは解決は無理。スクリプト含めて全部 cp932 で記載すれば解決できるかも
しれないが、やりたくない。
